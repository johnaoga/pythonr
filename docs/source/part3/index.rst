Objectifs
=========

À l'issue de cette partie, chaque étudiant.e sera capable de :

* Formuler des questions de recherche à partir d'un jeu de données
* Appliquer une méthodologie d'analyse exploratoire complète

* Produire un code Python et R de qualité (lisible, commenté, structuré)
* Présenter les résultats d'analyse de manière professionnelle

Méthodologie d'analyse exploratoire
""""""""""""""""""""""""""""""""""""

**1. Compréhension du contexte**

* Identifier la source des données
* Comprendre le domaine d'application
* Définir les objectifs de l'analyse
* Identifier les parties prenantes

**2. Collecte et import des données**

* Choisir le format approprié (CSV, Excel, JSON, base de données)
* Importer les données avec les outils adaptés (pandas, R base, tidyverse)
* Vérifier l'intégrité des données importées

**3. Exploration initiale**

* Dimensions du jeu de données (nombre de lignes, colonnes)
* Types de variables (numériques, catégorielles, dates)
* Premières observations (head, tail, sample)
* Statistiques descriptives de base

**4. Nettoyage des données**

* Identifier les valeurs manquantes (NA, NULL, NaN)
* Traiter les doublons
* Détecter les valeurs aberrantes (outliers)
* Corriger les incohérences (formats, encodage)
* Transformer les types de données si nécessaire

**5. Analyse exploratoire approfondie**

* Analyser la distribution des variables
* Étudier les relations entre variables (corrélations)
* Identifier des patterns et tendances
* Segmenter les données par groupes pertinents
* Formuler des hypothèses

**6. Visualisation**

* Créer des graphiques adaptés aux données
* Utiliser plusieurs types de visualisations complémentaires
* Personnaliser pour améliorer la lisibilité
* Mettre en évidence les insights principaux

**7. Synthèse et communication**

* Résumer les findings principaux
* Répondre aux questions de recherche
* Proposer des recommandations
* Documenter le processus et les résultats

Projet final 
=========================

.. note::
   Ce projet est en groupe et compte pour 30% de votre note finale.

Description du projet
"""""""""""""""""""""

**Objectif :** Mener une analyse exploratoire complète d'un jeu de données en utilisant Python ou R.

**Durée :** Plusieurs semaines (dates précises communiquées par l'enseignant)

**Thèmes proposés :**

Le professeur propose plusieurs thèmes/situations parmi lesquels vous devez choisir :

* Analyse de données éducatives (performances étudiantes, taux de réussite)
* Analyse de données de santé publique
* Analyse de comportement client (e-commerce, retail)
* Analyse de données environnementales (climat, pollution)
* Analyse de données sportives
* Analyse de données financières
* Ou tout autre thème validé par l'enseignant

**Choix du jeu de données :**

* Vous choisissez vous-même votre jeu de données selon le thème retenu
* Le dataset doit être pertinent pour le thème choisi
* Minimum 500 lignes et 5 colonnes
* Mélange de variables numériques et catégorielles
* Données réelles (pas de datasets jouets)

**Livrables :**

1. Code Python ou R
   
   * Fichiers .py ou .ipynb (Python)
   * Fichiers .R ou .Rmd (R)
   * Import, nettoyage et transformation des données
   * Analyses statistiques descriptives et exploratoires
   * Visualisations pertinentes
   * Code commenté, structuré et fonctionnel

2. **Rapport (maximum 10 pages)**
   
   * Description du jeu de données et du contexte
   * Questions de recherche formulées
   * Méthodologie d'analyse appliquée
   * Résultats et visualisations
   * Interprétation et conclusions
   * (Introduction, conclusion et références non comptées dans les 10 pages)


Critères d'évaluation
"""""""""""""""""""""

Le projet sera évalué selon 4 composantes  :

1. Code Python 

* Fonctionnalité et exécution sans erreur
* Structure et organisation
* Utilisation appropriée des structures de données
* Gestion des erreurs

2. Code R 

* Fonctionnalité et exécution sans erreur
* Structure et organisation
* Utilisation appropriée des objets R
* Gestion des erreurs

**3. Analyse exploratoire **

* Pertinence des questions de recherche
* Profondeur de l'analyse
* Justesse des statistiques calculées
* Interprétation des résultats

**4. Visualisations **

* Choix approprié des types de graphiques
* Qualité et lisibilité
* Personnalisation et esthétique
* Pertinence pour répondre aux questions



Conseils pour réussir
""""""""""""""""""""""

* **Organisez-vous en groupe** : Répartissez les tâches équitablement (Python, R, rapport, visualisations)
* **Commencez tôt** : Ne pas attendre la dernière minute
* **Validez votre thème et dataset** : Faites valider par l'enseignant avant de commencer
* **Choisissez bien vos données** : Un bon dataset = un bon projet
* **Posez des questions intéressantes** : Au-delà des statistiques de base
* **Documentez au fur et à mesure** : Plus facile que de tout faire à la fin
* **Testez régulièrement** : Assurez-vous que vos codes fonctionnent
* **Soignez les visualisations** : Elles communiquent vos résultats
* **Respectez la limite de 10 pages** : Synthèse et clarté sont essentielles
* **Relisez votre travail** : Vérifiez la cohérence entre code et rapport
* **N'hésitez pas à demander de l'aide** : Pendant les séances ou par email
